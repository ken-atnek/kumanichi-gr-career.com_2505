## UI実装ルールメモ

### 基本方針

- JSX 分岐で要素を増減させる前に、クラス切り替えで収まるかを確認する
- 見た目の調整はできるだけスタイル側で行う
- 不要になった分岐や配線は残さず整理する

### ハンバーガーやモーダルの考え方

- 開閉状態は `isOpen` のような分かりやすい state 名で管理する
- `aria-expanded` や `aria-label` を付ける
- 外側クリックで閉じるかどうかを先に決める

### クラス制御

- class 名の条件分岐が増える場合は `clsx` 導入を検討する
- 単一クラス指定だけなら無理に抽象化しない
