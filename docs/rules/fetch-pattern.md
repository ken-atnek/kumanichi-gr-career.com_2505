## データ取得パターン

`output: 'export'` を前提にする場合、ランタイムのサーバーサイド処理は使いにくい。  
データの出どころが固まるまでは、どこで何を読むかを先に決めてから実装する。

### 基本方針

- 表示用の固定データは、まずローカル定義か JSON 管理のどちらかを決める
- 変更頻度が高い文言や一覧は、あとから JSON に逃がせる形を意識する
- 管理画面や外部APIが絡む場合は、static export と両立できるか先に確認する

### 注意点

- `force-dynamic` は `output: 'export'` と競合するため使わない
- client component での fetch は、キャッシュ条件とエラー表示をセットで考える
- source of truth が複数にならないようにする
