## コーディング規約

### 基本方針

- 依頼がない限り、大きな構成変更をしない
- まずは既存コードへ自然に差し込める形を優先する
- 命名は用途が分かる形にする
- 将来的に JSON へ逃がす可能性がある文言は、直書きしすぎない

### React / Next.js 方針

- App Router 前提で進める
- 内部遷移は `Link` を使う
- 画像は原則 `next/image` を使う
- ページ専用パーツはページ名を先頭に付ける

#### 例

- `TopHero`
- `RecruitPageHero`
- `EntryCta`

### ファイル配置ルール

- 共通パーツは `src/components/common/` 配下
- ページ専用パーツは `src/components/{page}/` 配下
- コンポーネントと対応スタイルは同階層に置く

### 改行データ方針

- 改行が必要な文言は文字列配列で管理し、1要素を1行として描画する
- 句読点だけに依存した自動改行は行わない

### 外部リンク方針

- 外部遷移は `target="_blank"` と `rel="noopener noreferrer"` を忘れない
- 同じ記述が増える場合は共通コンポーネント化を検討する

### スタイル方針メモ

- 現時点では SCSS 未導入
- スタイル実装を本格化する段階で、CSS Modules 継続か SCSS 移行かを判断する
- SCSS 導入後は `docs/rules/scss-comment-rules.md` を基準にする
